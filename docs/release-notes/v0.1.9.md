# v0.1.9

## New Features

- **Clipboard OCR**: Image entries in the clipboard history now support OCR text extraction, making image content searchable via fuzzy match.
- **Clipboard search ranking**: Search results are now ranked by fuzzy match quality for better relevance.

## Security

- **Keychain disabled by default**: macOS Keychain integration is now off by default. API keys are stored as plaintext in `config.json`. To enable Keychain, set `"keychain": {"enabled": true}` in your config file.
- **Fixed secret leak on config save**: Previously, any settings change (e.g., toggling sound) could write real API keys back to `config.json` as plaintext, bypassing the Keychain sentinel replacement. All `save_config` calls now automatically protect secrets when Keychain is enabled.

## Bug Fixes

- **Recording stability**: Fixed recorder hang after stream close timeout; use `abort()` instead of `stop()` for faster stream cleanup.
- **Race conditions**: Resolved 5 race conditions in the recording pipeline for improved concurrency safety.

# v0.1.9 中文版

## 新功能

- **剪贴板 OCR**：剪贴板历史中的图片条目现在支持 OCR 文字提取，图片内容可通过模糊搜索查找。
- **剪贴板搜索排序**：搜索结果按模糊匹配质量排序，相关性更高的结果排在前面。

## 安全

- **Keychain 默认关闭**：macOS Keychain 集成现在默认关闭，API key 以明文存储在 `config.json` 中。如需启用 Keychain，请在配置文件中设置 `"keychain": {"enabled": true}`。
- **修复配置保存时的密钥泄露**：此前任何设置变更（如切换音效）都可能将真实 API key 以明文写回 `config.json`，绕过 Keychain 哨兵替换机制。现在所有 `save_config` 调用在 Keychain 启用时都会自动保护敏感字段。

## Bug 修复

- **录音稳定性**：修复音频流关闭超时后录音器卡死的问题；使用 `abort()` 替代 `stop()` 加速流关闭。
- **竞态条件**：修复录音管线中的 5 个竞态条件，提升并发安全性。
